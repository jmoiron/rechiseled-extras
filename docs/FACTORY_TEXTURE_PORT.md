# Factory/Technical Texture Port Plan

This document tracks how to migrate the classic Chisel `technical` and `technical/new` textures to the Rechiseled Extras 1.20/Fusion format.

## Goals

* Reproduce every Chisel "factory" texture so that it can be used by the modern Forge 1.20.1 port.
* Preserve CTM behavior by converting the historical CTM atlases (+ the vanilla `block.png` base) into Fusion 5×1 ("pieced") spritesheets or 4×1/1×4 directional sheets.
* Keep the original non-CTM textures (animated columns, single sprites, etc.) available under the new namespace.
* Keep new assets separate from the existing factory set by placing them under `assets/rechiseled_extras/textures/block/factory/technical/…` (and `…/technical/new` where appropriate).

## Tools

* `ctm-to-fusion.py` (already in repo). Run via `uv run ./ctm-to-fusion.py` or `python ctm-to-fusion.py`.
* `Pillow` is required by the script (already a dependency for other conversions).
* Basic `cp` / `python` file copies for assets that are not CTM-based.

## Output Layout

```
src/main/resources/assets/rechiseled_extras/textures/block/factory/
    technical/
        classic/<converted classic technical textures>
        new/<converted technical/new textures>
```

Using `classic` and `new` mirrors the Chisel layout and makes it obvious which assets came from which source set.

### Example command (standard CTM)

```
uv run ./ctm-to-fusion.py \
  Chisel/src/main/resources/assets/chisel/textures/blocks/technical/cautiontape-ctm.png \
  --outdir src/main/resources/assets/rechiseled_extras/textures/block/factory/technical/classic
```

This writes `.../factory/technical/classic/cautiontape.png` and `cautiontape.png.mcmeta`.

### Example command (vent top compositing)

```
uv run ./ctm-to-fusion.py \
  src/main/resources/assets/rechiseled_extras/textures/block/factory/vent-top.png \
  --composite Chisel/src/main/resources/assets/chisel/textures/blocks/technical/vent-top-ctm.png \
  --outdir src/main/resources/assets/rechiseled_extras/textures/block/factory/technical/classic \
  --file src/main/resources/assets/rechiseled_extras/textures/block/factory/technical/classic/vent_top_top.png
```

(Replace `--file` with whatever final name you want.)

## Automation Idea

To avoid hand-running dozens of commands, we can drop a helper script (e.g., `tools/convert_factory_textures.py`) that reads a YAML/JSON manifest. Each entry would contain:

```json
{
  "type": "ctm",
  "source": "Chisel/.../technical/cables-ctm.png",
  "base": "auto"
}
```

Supported entry types:

* `ctm`: simple `*-ctm` pair; script discovers sibling base.
* `ctm_composite`: CTM file lives elsewhere; pass `base` explicitly.
* `static_copy`: plain texture copy (optionally with `.mcmeta`).

The helper would call `ctm-to-fusion.py` for CTM entries and `shutil.copy2` for the others.

## Hooking Textures Into Blocks

Once textures exist in `factory/technical`, add blockstates/models/items for the new blocks:

1. Register new blocks/items in `ModBlocks`/`ModItems` (mirroring the existing factory entries).
2. Add `assets/rechiseled_extras/blockstates/factory_<name>.json` referencing `rechiseled_extras:block/factory/technical/<subdir>/<texture>`.
3. Add simple cube/cube_column models and item models pointing at the same textures.
4. Update language entries, loot tables, and recipes as usual.

For CTM-enabled ones, ensure their block models use Fusion syntax:

```json
{
  "parent": "rechiseled_extras:block/base_connecting",
  "textures": {
    "all": "rechiseled_extras:block/factory/technical/classic/cautiontape"
  }
}
```

The generated `.mcmeta` already encodes `{ "fusion": { "type": "pieced" } }`, so nothing else is needed.
